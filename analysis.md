# تحليل نظام نداء وانتظار المرضى

## نظرة عامة على النظام
نظام نداء وانتظار مريح متكامل باللغة العربية لمراكز طبية يتكون من 4 صفحات رئيسية.

## المتطلبات الرئيسية

### 1. صفحة الإدارة
- تعديل بيانات المركز والإعدادات العامة
- التحكم في سرعة نطق الصوت
- إدارة العيادات (إضافة، تعديل، حذف)
- اختيار أسلوب النطق (TTS أو ملفات MP3)
- إعدادات مسارات الصوتيات والميديا
- التحكم في شريط الأخبار
- نداء رقم معين على الشاشة
- تسجيل صوت مخصص

### 2. صفحة العرض
- شاشة عرض رئيسية للمرضى
- عرض اسم المركز والتاريخ والوقت
- كروت العيادات مع الأرقام الحالية
- QR Code للوصول للصفحة
- عرض الميديا التوعوية
- شريط أخبار متحرك
- تأثيرات بصرية عند النداء

### 3. صفحة التحكم
- نظام أمان بكلمة مرور للعيادات
- أزرار التحكم (التالي، السابق، تكرار، تصفير)
- نداء رقم معين
- عرض اسم مخصص

### 4. صفحة طباعة التذاكر
- طباعة تذاكر A4 بشبكة منتظمة
- كل تذكرة تحتوي على اسم العيادة والرقم
- معلومات فترة الانتظار

## التقنيات المستخدمة
- Firebase Realtime Database
- GitHub Pages
- Tailwind CSS
- Web Speech API (TTS)
- HTML5 Audio (MP3)

```
│   ├── center_name

│   └── news_ticker
├── clinics/

│   │   ├── name

```

## ملفات المشروع
- index.html (صفحة العرض)
- admin.html (صفحة الإدارة)
- control.html (صفحة التحكم)
- tickets.html (صفحة طباعة التذاكر)
- main.js (المنطق الرئيسي)
- firebase-config.js (إعدادات Firebase)
- audio-system.js (نظام الصوتيات)
- resources/ (مجلد الصوتيات والميديا)
